# 🌊 GuiaOrlaPE — Frontend Setup Guide

Bem-vindo(a) ao projeto **GuiaOrlaPE**!
Este guia foi criado para ajudar novos contribuidores a configurar rapidamente o ambiente de desenvolvimento do **frontend**.

---

## 📌 Sobre o Projeto

O GuiaOrlaPE é uma plataforma digital que conecta turistas a serviços locais em destinos litorâneos de Pernambuco.

O frontend foi desenvolvido utilizando:

* **Next.js**
* **React**
* **TypeScript**
* **TailwindCSS**

---

## ⚙️ Pré-requisitos

Antes de começar, você precisa ter instalado:

* [Node.js](https://nodejs.org/) (versão 18 ou superior)
* npm (geralmente já vem com o Node)
* Git
* (Opcional) Docker, caso queira rodar o projeto completo

---

## 📥 Clonando o Repositório

```bash
git clone https://github.com/seu-usuario/GuiaOrlaPE.git
cd GuiaOrlaPE/frontend/guiaorla-frontend
```

---

## 📦 Instalando Dependências

Dentro da pasta do frontend:

```bash
npm install
```

---

## 🚀 Rodando o Projeto

```bash
npm run dev
```

A aplicação estará disponível em:

👉 http://localhost:3000

---

## 🔗 Integração com o Backend

Por padrão, o frontend espera que o backend esteja rodando em:

```bash
http://localhost:5000
```

Se necessário, ajuste o arquivo:

📁 `src/services/api.ts`

```ts
export const API_URL = "http://localhost:5000";
```

---

## 🐳 Rodando com Docker (Opcional)

Se quiser subir tudo com Docker:

Na raiz do projeto:

```bash
docker-compose up --build
```

Isso irá subir:

* PostgreSQL
* Backend (.NET)
* Frontend (Next.js)

---

## 📁 Estrutura do Projeto

```bash
src/
│
├── app/            # Rotas (App Router - Next.js)
├── components/     # Componentes reutilizáveis
├── services/       # Comunicação com API
├── styles/         # Estilos globais
```

---

## 🧠 Boas Práticas

✔️ Use componentes reutilizáveis
✔️ Evite lógica pesada dentro das páginas
✔️ Centralize chamadas de API em `/services`
✔️ Utilize Tailwind para estilização
✔️ Nomeie componentes de forma clara

---

## ✨ Padrões de Código

* Tipagem com TypeScript obrigatória
* Componentes funcionais (React)
* Organização por responsabilidade

---

## 🧪 Testando a Integração

Certifique-se de que:

1. O backend está rodando
2. A API responde corretamente
3. Os dados aparecem na interface

---

## 📌 Dicas para Desenvolvimento
